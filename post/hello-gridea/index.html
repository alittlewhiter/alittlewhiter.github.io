Coding/GitHub都提供免费的pages服务，即提供静态网页托管和演示服务，可以使用 Coding Pages或GitHub Pages 托管博客、项目官网等静态网页，也可以部署开源博客、CMS 等动态网页。

选择合适的pages 服务

搭建博客之前，首先要选择合适pages服务，两个网站托管的免费博客都可以自定义域名，也可以使用他们提供的免费域名。他们之间的区别如下：

不过两个pages服务都有一些资源的限制：
Coding pages服务的限制如下：

  • Coding Pages 部署来源的大小不超过 100M
  • 每小时最多可部署 10 次

GitHub Pages 站点受到以下使用限制的约束：

  • GitHub Pages 源仓库建议的限制为 1GB。
  • 发布的 GitHub Pages 站点不得超过 1 GB。
  • GitHub Pages 站点的带宽限制为每月 100GB。
  • GitHub Pages 站点的限制为每小时 10 次构建。

安装Gridea

官方网站：Gridea
点击即可访问、下载、安装、顺便逛逛官网主题页面。。。

安装之后点击打开，进入设置->源文件目录，设置一个自己本地的目录

配置Coding pages 服务

首先注册并登陆Coding（已与腾讯开发者平台合并）

  • 新建项目，选择项目模板（DevOps项目/代码托管项目）

  • 填写项目基本信息：项目名称、项目标识（与url相关），设置Git项目仓库

  • 打开项目设置->项目与成员->功能开关->打开构建与部属等功能，回到项目主界面，点击构建与部署->静态网站

  • 经第一步实名认证后，可直接点击 立即发布静态网站，输入网站名称点击保存即可。

  • 此时会给出一个可访问的域名（如http://ey3un0.coding-pages.com），点击立即部署，即出现第一次部署记录。

  • 接下来，点击用户头像->个人设置-> 访问令牌 -> 新建令牌 -> 输入令牌描述、勾选权限-> 确认后得到令牌用户名 & [令牌](Ko no dio da! 233)。（注意访问令牌仅显示一次，安全起见合理保存）

  • 打开Gridea（v0.92），

  • 点击检测远程连接，成功后点击保存，同步即可。

  • 浏览器输入自己的Page网址。

Github Page配置

必须：确保已经安装过 Git 和 Gridea
注意：同步会覆盖掉仓库的文件，若仓库内有需要备份的，请手动备份

首先需要一个 Github 账号，如果没有的话，快去注册一个吧！

登录 Github，然后进入到 这里 创建一个 Token，勾选上 repo 的相关权限即可，下面是对应的截图示例：

第一次使用它需要你进行应用的初始化配置，才能让他和 Github Pages 连接，配置很简单，可以参考下面的方法进行配置：

  • 域名：Github Pages 对应的域名（例如：https://alittlewhiter.github.io ）
  • 仓库：你的静态文件存放的仓库（例如：alittlewhiter.github.io）
  • 分支：你的 Github Pages 对应的分支（例如：master）
  • 用户名：仓库所属用户的用户名（通常就是你的 Github 账号的昵称）
  • 邮箱：Git 推送时使用的邮箱（通常就是你的 Github 账号的邮箱）
  • Token: Git 推送时需要的 Token，用来向 Github 提交构建后的文件 Github personal access tokens
  • CNAME: 可通过这个选项配置你自己的域名（例如：moyu.life）

Tip: 创建 Personal access tokens 的时候，只需要赋予 repo 的权限即可，这样最大程度减少 Token 出现泄漏后可能造成的安全风险。

另外！注意：

Token字符串不能存储在github的repo中，经过测试，一旦提交的文件中包含这个token字符串，那么github就会自动删除这个token -_-! 错了很久才明白过来，创建的Personal Access Token总是自动消失，导致莫名其妙同步失败。。
